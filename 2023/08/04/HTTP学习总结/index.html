HTTP协议学习总结

发表于2023-08-04|更新于2023-08-07
|阅读量:

url和uri

  • URI. 统一资源标识符（HTTP协议使用URI让客户端定位到资源）

uri格式：协议+登录信息（认证）+服务器地址+服务器端口号+带层次的文件路径+查询字符串+片段标识符

  • URL.统一资源定位符

http协议

  • 用于客户端和服务器之间的通信。
  • 通过请求和响应的交换达成通信，是无状态协议（不 保留之前一切的请求或响应报文的信息，使用cookie进行状态管理）

请求报文

由请求方法、请求URI、协议版本、可选的请求首部字段和内容实体构成 的。

http方法
get 获取资源
POST 传输实体主体
PUT 传输文件(存在安全问题)
HEAD 获得报文首部
DELETE 删除文件（不带验证机制，一般不使用）
OPTIONS 询问支持的方法

HTTP状态码

表示客户端HTTP请求的返回结果
1XX 正在处理
2XX 成功
3XX 重定向
4XX 客户端错误
5XX 服务器错误
  • 204:返回的响应报文中不含实体的主 体部分
  • 206：部分资源响应
  • 301: 永久性重定向，请求的资源已被分配了新的URI，以后应使用资源现在 所指的URI
  • 302 ：临时性重定向，请求的资源已被分配了新的URI，希望用户（本次）能 使用新的URI访问
  • 303：有另一个URI，应使用GET方法定向获取请求 的资源
  • 304: 服务器端允许请求访问资源，但请求未满足条件的情况
  • 307: 临时重定向
  • 400: 请求报文中存在语法错误
  • 401:发送的请求需要有通过HTTP认证的认证信息（首次401，会弹出认证用的对话窗口）
  • 403：访问被服务器拒绝了（访问权限问题）
  • 404: 服务器上无法找到请求的资源
  • 500:服务器端在执行请求时发生了错误
  • 503：服务器无法处理，可能暂时处于超负载或正在进行停机维护

http首部字段类型

有四种类型：

  • 通用首部字段
  • 请求首部字段
  • 响应首部字段

通用首部字段
Cache-Control 控制缓存的行为
Connection 逐跳首部、连接的管理创建报文的日期时间报文指令
Date 报文末端的首部一览
Pragma 指定报文主体的传输编码方式
Trailer 升级为其他协议
Transfer-Encoding 代理服务器的相关信息
Upgrade 升级为其他协议
Via 代理服务器的相关信息
Warning 错误通知

响应首部字段
Accept-Ranges 是否接受字节范围请求
Age 推算资源创建经过时间
ETag 资源的匹配信息
Location 令客户端重定向至指定URI
Proxy-Authenticate 代理服务器对客户端的认证信息
Retry-After 对再次发起请求的时机要求
Server HTTP服务器的安装信息
Via 代理服务器的相关信息
WWW-Authenticate 服务器对客户端的认证信息

实体首部字段
Allow 资源可支持的HTTP方法
Content-Encoding 实体主体适用的编码方式
Content-Language 实体主体的自然语言
Content-Length 实体主体的大小（单位：字节）
Content-Location 替代对应资源的URI
Content-MD5 实体主体的报文摘要
Content-Range Content-Type Expires 实体主体的位置范围实体主体的媒体类型实体主体过期的日期时间
Last-Modified 资源的最后修改日期时间

请求首部字段
Accept 用户代理可处理的媒体类型
Accept-Charset 优先的字符集
Accept-Encoding 优先的内容编码
Accept-Language 优先的语言（自然语言）
Authorization Web认证信息

Session管理及Cookie应用

过程：

  1. 发送登陆信息：客户端把用户ID和密码等登录信息 发送给服务器。 用HTTPS通信来进行HTML表单画面的显示和 用户输入数据的发送。
  2. 发sessionID：服务器发放Session ID给客户端，用以识别用户。 然后把用户的认证状态与Session ID绑定后记录在服务器端。
  3. 保存为Cookie：客户端接收到从服务器端发来的Session ID后，作为Cookie保存在本地。
  4. 用Cookie请求：下次向服务器发送请求时， 自动发送Cookie，所以Session ID也随 之发送到服务器。服务器端通过收到的Session ID识别用户和其认证状态。
