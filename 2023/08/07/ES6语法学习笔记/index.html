promise

优雅地解决异步回调，避免回调地狱

js处理异步的两种方式:回调函数 promise

场景：要在一个异步函数之后执行第二个异步操作，第三个异步操作在第二个异步操作后执行，这样就会形成很多层的嵌套，就是回调地狱

使用promise，链式结构避免多层嵌套，向后延伸而不是向里

promise

  • 一个构造函数
  • 参数是一个函数，改函数有两个参数：reslove，reject
  • 三种状态：pedding进行中，fulfilled：成功，rejected失败
  • pedding->fulfilled/rejected 不可逆
  • Promise对象上的方法：resolve，reject，all（都返回成功时执行），race，reject，any(存在一个成功时执行)，race（有一个有结果执行（无论成功失败））
  • Promise的原型对象上的方法： then、catch、finally

api

then：

  • 处理promise状态改变后的操作，同步执行
  • then中接受两个回调函数作为参数，状态发生变化的时候，执行内部对应的回调函数
  • then的第一个回调函数是处理promise实例成功状态的,函数的参数就是当前成功promise实例的值，then的第二个回调函数是处理promise实例是失败状态的，函数的参数就是当前失败的promise实例的值
  • then的执行是同步的，但是then中的回调函数的执行是一定异步的,then的同步执行就是为了准备两个回调函数

catch：

  • 当状态变为失败的时候，执行catch中的回调函数，catch方法是同步的，catch中的回调函数的是异步的
  • then或者catch中的回调函数只要报错（以上任何一个步骤出错）、2. then或者catch中的回调函数中返回失败promis对象

finally

  • 无论成功还是失败，最后都会执行，执行一些最后的清理工作
